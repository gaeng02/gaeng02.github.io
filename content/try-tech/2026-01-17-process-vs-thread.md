# 프로세스 vs 스레드

운영체제에서 프로세스와 스레드는 중요한 개념입니다.

## 프로세스 (Process)

프로세스는 **실행 중인 프로그램**입니다.

### 특징

- 독립적인 메모리 공간
- 프로세스 간 통신(IPC) 필요
- 컨텍스트 스위칭 비용이 큼

### 프로세스 구조

```
프로세스
├── 코드 영역
├── 데이터 영역
├── 힙 영역
└── 스택 영역
```

## 스레드 (Thread)

스레드는 **프로세스 내의 실행 단위**입니다.

### 특징

- 같은 프로세스 내 스레드들은 메모리 공유
- 스레드 간 통신이 쉬움
- 컨텍스트 스위칭 비용이 작음

### 멀티스레딩

하나의 프로세스가 여러 스레드를 가질 수 있습니다.

## 비교

| 항목 | 프로세스 | 스레드 |
|------|---------|--------|
| 메모리 | 독립적 | 공유 |
| 통신 | IPC 필요 | 쉬움 |
| 오버헤드 | 큼 | 작음 |
| 안정성 | 높음 | 낮음 |

## 언제 무엇을 사용할까?

- **프로세스**: 독립적인 작업, 높은 안정성 필요
- **스레드**: 공유 데이터가 많고, 빠른 통신 필요

## 결론

프로세스와 스레드는 각각의 장단점이 있으므로, 상황에 맞게 선택해야 합니다.
